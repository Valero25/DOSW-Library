🔐 SEGURIDAD - JWT + RBAC
═══════════════════════════════════════════════════════════════════

Autenticación:
  ✅ JWT (JSON Web Tokens) con HMAC-SHA384
  ✅ Token expiration: 1 hora (configurable)
  ✅ Stateless (sin sesiones)
  ✅ BCrypt para hash de contraseñas

Autorización (Role-Based Access Control):
  ├─ /auth/** → PÚBLICO (sin autenticación)
  ├─ GET /api/books → Autenticado
  ├─ POST/PUT/DELETE /api/books → LIBRARIAN only
  ├─ GET /api/users → LIBRARIAN only
  ├─ /api/loans → Autenticado
  └─ Swagger UI → Público (sin token)

🎓 APRENDIZAJES CLAVE IMPLEMENTADOS
═══════════════════════════════════════════════════════════════════

1. PATRÓN STRATEGY + SPRING PROFILES
   → Permite cambiar persistencia sin modificar servicios

2. DUAL PERSISTENCE
   → PostgreSQL (relational) + MongoDB (nonrelational) coexisten
   → Selección por perfil en tiempo de boot

3. INYECCIÓN AUTOMÁTICA
   → Spring elige implementación según @Profile activo
   → Limpio, mantenible, testeable

4. DATA INITIALIZATION
   → CommandLineRunner ejecuta al startup
   → Carga datos de prueba automáticamente
   → Usa PasswordEncoder para seguridad

5. JWT STATELESS
   → Token generado en login
   → Validado en cada request
   → No requiere sesiones en servidor
